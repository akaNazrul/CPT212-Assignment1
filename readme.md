## 📁 Project Structure
The repository contains the required source code, output data, and visualization scripts corresponding to the assignment parts:

* **Main.java:** The entry point of the program. Handles the generation of random $-digit numbers, initiates the multiplications, and manages the execution timers/operation counters.
* **SimpleMultiplication.java:** Contains the implementation for **Part 1**. Handles multiplication by multiplying each digit and storing partial products and carriers, eventually summing them.
* **KaratsubaModified.java:** Contains the implementation for **Part 2**. Handles the modified Karatsuba multiplication algorithm equipped with primitive operation counters.
* **graph.py:** A Python script used to parse the counter data and plot the comparison graph (Number of Operations vs. $).
* **mult_ops.csv:** Output CSV file generated by the Java application containing the collected operation counts across different $ sizes.

## ✨ Features Implemented
### Part 1: Simple Multiplication
* Exact implementation of the standard multiplication process using Java SE.
* Prints partial products and carriers at every multiplication step (demonstrated accurately for small numbers).
* Robust enough to handle exceedingly large numbers (e.g.,  = 10,000$ digits).
* Instrumentations built-in to count primitive operations (assignments, additions, etc.).

### Part 2: Karatsuba Algorithm
* Modified Karatsuba implementation optimized for extremely large scale numbers.
* Built-in primitive operation tracking.
* Automated CSV export (mult_ops.csv) for benchmarking operation counts against the Simple Multiplication implementation over varying scales of $.

## 🚀 Setup & Execution Instructions

### Prerequisites
* **Java Development Kit (JDK)** (Java SE only, strictly no external APIs).
* **Python 3.x** (Optional) Requires matplotlib and pandas for plotting graphs.

### 1. Compile the Java Programs
Navigate your terminal to the project directory and compile all Java files:
`ash
javac *.java
`

### 2. Run the Java Program
Launch the Main class. This will perform the sample multiplications and generate the CSV file with primitive operation counts.
`ash
java Main
`

### 3. Generate the Complexity Graph
After running the java application (which populates mult_ops.csv), generate the complexity comparison chart:
`ash
pip install matplotlib pandas

`
This will plot "Number of Operations vs. $" for both Simple and Karatsuba algorithms.

## 🧠 Complexity Analysis Summary
According to the empirical results generated in the benchmarks:
* **Simple Multiplication Algorithm:** Follows an (N^2)$ time complexity. As $ increases, the number of primitive operations quadrates since every digit must be multiplied against every other digit.
* **Karatsuba Algorithm:** Demonstrates a more optimal (N^{\log_2 3}) \approx O(N^{1.58})$ time complexity. For very large values of $, Karatsuba consistently registers significantly fewer primitive operations.
